Отчёт технического плана по С++

За 2022-05-25 по 2022-06-07

За данный период изучал CMake. CMake -- инструмент для автоматизации сборки приложения, тестирования и т.д. Упрощает процесс подключения библиотек (по крайней мере на Unix). Было изучено:

  1. Сборка простого приложения. Практика
  2. Сборка приложения с user-defined библиотеками и без. (add_subdirectory)Практика
  3. Сборка приложения с библиотеками со стороны (find_library, find_package)Практика
  4. Деплой и сборка установщика.Практика
  5. Установка либ через гитхаб. Практика

Плюсы и минусы CMake:

  1. + -- Значительно упрощает подключение сторонних либ.
  2. + -- Кросс платформенность (но с нюансами).
  3. + -- Задает хорошую структуризацию проекта.
  1. - -- для подключения библиотек, нужно понимать строение этих библиотек и их конфиг
  2. - -- неудобно собирать проект на винде без Visual Studio Code
  3. - -- без видеоуроков тяжеловато по-началу изучать.

Обучение не логгировалось :)
