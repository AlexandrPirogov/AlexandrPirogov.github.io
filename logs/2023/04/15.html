Изучил системный уровень ввода/вывода в UNIX. Все устройства ввода/вывода смоделированы в виде файлов в Linux. B ввод, и вывод выполняются путем чтения и записи в соответствующий файл. Такое простое отображение дает единообразное выполнения операций над файлами:

  • Открытия файла. При открытия файла, посылается запрос ядру. Ядро возвращает небольшое неотрицательное число -- дескриптор файла -- идентификатор открытого фалйа.
  • Изменения текущей позиции файла. Изначально позиция равна 0, т.е. начало файла.
  • Чтение и запись в файл. Производится чтения или запись n байт.
  • Закрытия файла. Посылается запрос ядру, в ответ на что ядро освобождает созданные при открытии файла структуры.

    • Файлы разделяются на три основных типа:

  • Обычный файл -- это могут быть текстовые или бинарные файлы.
  • Каталог -- файл, содержащий ссылки, каждая из которых отражает имя файла в физический файл.
  • Сокерт -- файл используется для взаимодействий с другим процессом по сети.

    • Файлы в Linux можно использовать совместно разными способами. Для этого используются следующие структуры данных:

  • Таблица дескрипторов -- структура данных, которая создается внутри каждого процесса, представленная в виде таблица, элементы которой индексируются дескрипторами открытых файлов. Для каждого подобного дексриптора имеется свой элемент в таблице открытых файлов.
  • Таблица открытых файлов -- общая для всех процессов структура данных, представленная в виде таблицы, которая содержит текущую позицию в файле и счетчик ссылок -- кол-во дескрипторов, ссылающихся на данных момент на этот файл.
  • Таблица виртуальных узлов общрая для всех процессов структура данных, которая хранит в себе метаинформацию о файле.

    • Также изучил основные концепты конкурентного программирования. Конкурентное программирование может быть реализованы тремя способами:

  • конкурентное программирование на уровне процессоров
  • мультиплексирование ввода/вывода
  • конкурентное программирование с потоками выполнения
