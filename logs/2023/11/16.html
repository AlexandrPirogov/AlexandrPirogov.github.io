Общая рефлексия

За данный период изучал и практиковал взаимодействие с API k8s. Для упрощения доступа можно сконфигурировать прокси-сервер:
> kubectl proxy --port 8080
По адресу http://localhost:8080 можно получить JSON документ, который описывает все доступные операции по ключу path.

Взаимодействие с API k8s происходило через python и curl. В основном воспроизводил операции, которые применял ранее в императивном стиле через kubectl -- создание, управление ресурсами и их просмотр. C помощью API можно создать интересные пет-проекты для автоматизации взаимодействия с k8s.
