Архитектура k8s

Основные компоненты

Кластер

Кластер — это набор компьютеров, хранилищ данных и сетевых ресурсов, с помощью которых Kubernetes выполняет различные задачи в вашей системе. Стоит отметить, что система может состоять из нескольких кластеров.

Узел/нода

Узел - отдельный физический/виртуальный компьютер.

Под

Под - единица работы в k8s, содержащий один и более контейнеров. Поды всегда работают совместно, т.е на одном компьютере. Все контейнеры внутри пода имеют один и тот же IP-адрес и пространство портов. Контейнеры внутри пода общаются между собой через локальный сервер или с помощь IPCP. Все контейнеры имеют доступ к общему локальному хранилищу данных узла, на котором находится под.

Контроллеры репликации и наборы реплик

Контроллеры репликации и наборы реплик предназначены для управления группами подов, выбранных с помощью селекторов меток. Они гарантируют, что определенное количество экземпляров подов всегда находится в рабочем состоянии. Контроллеры репликации проверяют принадлежность к группе по одному имени. Наборы реплик позволяют указать несколько значений.

Сервисы

Сервисы применяются для предоставления клиентам определенных функций, например, предоставить доступ к внешним ресурсам и экзмеплярам подов, которыми управляют на уровне виртуальных IP-адресов. Иначе говоря, точка соединения с подом.

Kubelet

Kubelet - агент, работающий на каждом узле, дающий возможность ноде взаимодействовать с K8s master.

Controle Pane/Master

Control Pane - набор контейнеров, которые управляют кластером. Включает в себя API-server, scheduler, controller manager, etcd и т.п.

etcd

etcd - распределенное key-value хранилище.

API

API - способ общения с кластером.

Scheduler

Scheduler - контейнер управляет располложением контейнеров в кластере.

Controller manager

Controller manager следит за состоянием всего кластера.
